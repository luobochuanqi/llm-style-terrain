## 📋 模型概览

### 核心架构
- **类型**: cVAE (Conditional Variational Autoencoder)
- **条件化**: FiLM (Feature-wise Linear Modulation)
- **分辨率**: 支持 256×256 和 512×512
- **应用场景**: 地形高度图风格迁移 (丹霞 ↔ 喀斯特)

---

## 🔧 模型参数

### 256 分辨率配置

| 组件 | 参数量 | 说明 |
|------|--------|------|
| **Encoder** | ~8.5M | 5 层 downsampling (256→8) |
| **FiLM Decoder** | ~9.5M | 5 层上采样 + FiLM 调制 |
| **Condition Normalizer** | ~1K | 对数变换 + 标准化 |
| **总计** | **~18.4M** | 可训练参数 |

### 512 分辨率配置

| 组件 | 参数量 | 说明 |
|------|--------|------|
| **Encoder** | ~15.2M | 7 层 downsampling (512→4) |
| **FiLM Decoder** | ~15.8M | 6 层上采样 + FiLM 调制 |
| **Condition Normalizer** | ~1K | 对数变换 + 标准化 |
| **总计** | **~31.0M** | 可训练参数 |

### 详细架构

#### Encoder (512 分辨率)
```
Input: (1, 512, 512)
→ Conv2d(1, 32, 4, 2, 1) + BN + ReLU        → (32, 256, 256)
→ Conv2d(32, 64, 4, 2, 1) + BN + ReLU       → (64, 128, 128)
→ Conv2d(64, 128, 4, 2, 1) + BN + ReLU      → (128, 64, 64)
→ Conv2d(128, 256, 4, 2, 1) + BN + ReLU     → (256, 32, 32)
→ Conv2d(256, 256, 4, 2, 1) + BN + ReLU     → (256, 16, 16)
→ Conv2d(256, 512, 4, 2, 1) + BN + ReLU     → (512, 8, 8)
→ Conv2d(512, 512, 4, 2, 1) + BN + ReLU     → (512, 4, 4)
→ Flatten → Linear(512×4×4, 256)            → mu, logvar (各 256 维)
```

#### FiLM Decoder (512 分辨率)
```
Input: z (256) + condition (3)
→ Linear(256, 512×4×4) → Reshape(512, 4, 4)
→ [ConvTranspose2d + BN + ReLU + FiLM] × 6 层
   512×4×4 → 512×8×8 → 512×16×16 → 256×32×32 → 
   128×64×64 → 64×128×128 → 32×256×256
→ ConvTranspose2d(32, 1, 4, 2, 1) + Sigmoid  → (1, 512, 512)
```

## 📥 输入输出

### 输入

| 字段 | 形状 | 类型 | 说明 |
|------|------|------|------|
| **高度图** | `(batch, 1, 512, 512)` | float32 [0,1] | 归一化高度图 (黑=谷，白=峰) |
| **条件向量** | `(batch, 3)` | float32 | [S_score, R_score, C_score] |

#### 条件向量说明
- **S_score (Sharpness)**: 锐度 (0-10) - 山峰尖锐程度
- **R_score (Ruggedness)**: 粗糙度 (0-10) - 地形起伏程度
- **C_score (Complexity)**: 复杂度 (0-10) - 细节丰富程度

**预处理**:
```python
# C_score 对数变换 (解决严重右偏)
C_score_log = log(C_score + 1e-5)

# 标准化 (基于训练集统计)
S_score: mean=2.56, std=1.68
R_score: mean=4.04, std=1.89
C_score_log: mean≈0, std≈1.5
```

### 输出

| 字段 | 形状 | 类型 | 说明 |
|------|------|------|------|
| **重建高度图** | `(batch, 1, 512, 512)` | float32 [0,1] | 生成的地形高度图 |
| **隐变量均值** | `(batch, 256)` | float32 | 隐空间分布均值 μ |
| **隐变量方差** | `(batch, 256)` | float32 | 隐空间对数方差 log(σ²) |

## 🎯 训练过程

### 训练配置 (Phase 1 优化)

| 参数 | 256 配置 | 512 配置 |
|------|----------|----------|
| **Batch Size** | 8 | 4 |
| **初始学习率** | 1e-4 | 1e-4 |
| **学习率调度** | Cosine (1e-4→1e-6) | Cosine (1e-4→1e-6) |
| **Beta (KL 权重)** | 2.0 | 2.0 |
| **Beta Warmup** | 150 epochs | 150 epochs |
| **总 Epochs** | 200 | 200 |
| **梯度损失权重** | 0.2 | 0.2 |
| **早停** | 禁用 | 禁用 |

### 损失函数

```python
Total Loss = L_recon + β × L_KL + λ × L_grad

其中:
- L_recon = MSE(reconstruction, target)           # 重建损失
- L_KL = -0.5 × Σ(1 + log(σ²) - μ² - σ²)         # KL 散度
- L_grad = MSE(∇x_pred, ∇x_target) + 
           MSE(∇y_pred, ∇y_target)               # 梯度损失 (Sobel)
- β = 2.0 (warmup 150 epochs 从 0 线性增加)
- λ = 0.2
```

### 训练数据

| 划分 | 样本数 | 地貌类型 |
|------|--------|----------|
| **训练集** | 87 | 丹霞 67 + 喀斯特 58 (加权采样) |
| **验证集** | 19 | 丹霞 10 + 喀斯特 9 |
| **测试集** | 19 | 丹霞 10 + 喀斯特 9 |

**数据增强**:
- ✅ 水平翻转 (p=0.5)
- ✅ 90° 旋转 (k∈{0,1,2,3})
- ✅ 小角度旋转 (±5°)
- ❌ 垂直翻转 (禁止！山会变坑)

### 训练时间

| 模式 | Epochs | 256 耗时 | 512 耗时 |
|------|--------|----------|----------|
| **Debug** | 10 | 0.5 分钟 | 0.5 分钟 |

## ⚠️ 显著问题

### 1. **轻度过拟合** (已观察)

**现象**:
- Epoch 95 后 Val Loss 缓慢上升 (5481 → 6214)
- Train Loss 持续下降但 Val Loss 持平

**原因**:
- 数据集较小 (87 训练样本)
- 模型容量较大 (31M 参数)
- 训练 200 epochs 可能过长

**当前缓解措施**:
- ✅ Beta-VAE (β=2.0) 增加正则化
- ✅ 加权采样平衡类别
- ✅ 安全数据增强 (翻转/旋转)

**建议改进**:
1. 增加正则化 (weight_decay 从 1e-5 → 1e-4)
2. 使用 epoch 95 的 checkpoint (最佳验证损失)
3. 实施 U-Net 架构 (skip connections 减少过拟合风险)

---

### 2. **细节模糊** (VAE 固有问题)

**现象**:
- 生成结果偏平滑，缺乏高频细节
- 山脊和峡谷边缘不够锐利

**原因**:
- MSE 损失倾向于平均化
- VAE 隐变量正则化导致信息丢失

**当前缓解措施**:
- ✅ 梯度损失 (Sobel 边缘检测) λ=0.2
- ✅ 小角度旋转增强 (±5°)

**建议改进**:
1. **U-Net 架构** (skip connections 保留高频) - 预期提升 +40%
2. **VQ-VAE** (离散隐空间) - 预期提升 +50%
3. **Perceptual Loss** (VGG 特征) - 额外提升 +10-15%

---

### 3. **固定花纹/重复模式**

**现象**:
- 某些区域出现重复纹理
- 缺乏自然地形的随机性

**原因**:
- 数据增强不足
- 训练早期停止 (旧配置 epoch 103 停止)
- Beta 增加过快

**当前缓解措施**:
- ✅ 禁用早停，完成 200 epochs
- ✅ Beta warmup 延长至 150 epochs
- ✅ Cosine LR 后期精细收敛

**建议改进**:
1. 增加亮度/对比度增强 (±10%)
2. 添加轻微高斯噪声 (σ=0.02)
3. U-Net 架构增加多样性

---

### 4. **C_score 控制不准确**

**现象**:
- 改变 C_score 对生成结果影响不明显
- S_score 和 R_score 控制效果更好

**原因**:
- C_score 严重右偏 (80% 样本 <2.0)
- 对数变换可能不足

**当前缓解措施**:
- ✅ C_score 对数变换：log(C + 1e-5)
- ✅ 条件归一化由模型内部处理

**建议改进**:
1. 分位数归一化 (替代标准归一化)
2. 增加 C_score 条件权重
3. 条件 ablation 研究验证

---

### 5. **隐空间维度可能不足**

**现象**:
- KL 散度偏低 (150-600) 相对于重建损失 (4000-7000)
- 可能存在隐变量未充分利用

**原因**:
- latent_dim=256 可能不足以捕捉 512 分辨率细节
- Beta 较高 (2.0) 抑制 KL 散度

**当前状态**:
- ✅ KL/Recon 比例 ~14% (健康范围 10-20%)

**建议改进**:
1. 增加 latent_dim 至 512 (512 分辨率)
2. 降低 Beta 至 1.5 (优先重建质量)
3. KL 散度 ablation 研究
